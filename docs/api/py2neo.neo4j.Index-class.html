Class Index

source code

   object --+    
            |    
rest.Resource --+
                |
               Index

Represents an Index within a Neo4j database instance identified by a URI and/or a template URI. With a nod to Java generics, an Index instance may hold either Nodes or Relationships by supplying the appropriate class directly to the constructor. For example:

>>> from py2neo import neo4j
>>> Index(neo4j.Node, "http://localhost:7474/db/data/index/node/index1")
Index<Node>(u'http://localhost:7474/db/data/index/node/index1')

By default, every Index write operation will execute immediately but these may also be grouped into a batch in order to reduce HTTP calls. To create a batch, simply issue a batch_start() call, followed by the required add and remove calls and finally calling batch_submit(). A batch may be discarded at any time using batch_discard().

remove(self, entity)

source code 

Removes the given entity from this Index. The entity must have been retreived from an Index search and therefore contain an _index_entry_uri property. If a batch has been started, this operation will not be submitted until submit_batch() has been called.

Parameters:
  • entity - the resource to remove from the Index

search(self, key, value)

source code 

Searches the current Index for items matching the supplied key/value pair; each pair may return zero or more matching items. The value parameter may also be a list allowing a batch search to be performed matching the specified key with each value in turn. In batch mode a list of lists will be returned, each top-level item corresponding to the value passed into the same position.
